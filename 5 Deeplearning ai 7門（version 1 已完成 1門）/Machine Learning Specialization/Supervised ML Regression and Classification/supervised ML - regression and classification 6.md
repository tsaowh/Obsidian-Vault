- ### gradient descent in practice
	- #### choosing the learning rate
	- ![[截圖 2026-02-17 晚上8.06.17.png]]
		- 學習率（learning rate, α）對梯度下降法是否順利收斂至關重要
			- α 太小 → 收斂很慢
			- α 太大 → 可能震盪甚至發散
			- 若 α 很小但 cost 仍不穩定 → 可能是程式有 bug
		- 如何辨識 gradient descent 出問題
			- Cost 有時上升、有時下降
				- 學習率太大
				- 更新步伐過大，跨過最小值
				- 在谷底左右來回跳動（overshooting）
			- Cost 持續上升
				- cost 每次 iteration 都增加
				- 學習率太大
				- 程式實作錯誤
		- 重要 Debug 技巧
			- 若 α 設得非常小 cost 仍然不穩定
			- 則 大機率是程式實作錯誤
	- ![[截圖 2026-02-17 晚上8.06.36.png]]
		- α 太大 vs α 太小
			- α 太大
				- 更新步伐太大
					- 每次更新都跨過最小值
				- cost 震盪或上升
					- 不斷 overshoot
					- 造成震盪或發散
			- α 太小
				- 更新步伐太小
				- cost 平穩下降
					- 下降速度非常慢
					- 需要很多 iterations 才接近最小值
		- 實務上，如何選擇 α（learning rate tuning）
			- log-scale search
				- 不是線性增加
				- 而是指數式增加
					- eg 每次大約放大 3 倍
			- 找到 不會震盪、收斂速度快 的最大 α
	- #### feature engineering
	- ![[截圖 2026-02-17 晚上8.45.59.png]]
		- 模型本身不是最重要的瓶頸，特徵才是
			- 即使是簡單的線性模型，只要特徵設計得好，也能有很不錯的效果
			- 反之，再複雜的模型，如果輸入特徵沒有抓到問題本質，也很難學好
		- **特徵工程的核心動作**：
			- 用直覺幫助模型（不是隨機亂試，而是有思考的設計）
				- 轉換特徵（例如平方、對數）
				- 組合特徵（例如相乘、比例）
					- eg
						- 地寬對價格有影響
						- 地深對價格有影響
						- but 房價往往和土地總面積關係更密切
				- 利用問題的 領域知識（domain knowledge） 設計新變數
			- 好的 特徵 可以：
				- 降低模型複雜度
				- 加快訓練速度
				- 提升準確率
				- 增強可解釋性
		- 現代深度學習 某種程度上會 自動學習特徵
	- #### polynomial regression
	- ![[截圖 2026-02-17 晚上9.24.19.png]]
		- Polynomial Regression（多項式回歸）
			- 將「線性回歸」透過 特徵工程（feature engineering）擴展成 可以擬合曲線的模型。
			- 依然是**線性模型（linear in parameters）**，只是：
				- 特徵變成：
					- $x$
					- $x^2$
					- $x^3$
			- 此模型對「參數 w」是 線性的，對「x」是 非線性的
		- 為什麼需要多項式？
			- 模型表達能力（Model Capacity）
				- 一次：表達能力低，可能 underfit
				- 二次：有彎曲能力，but 可能「往下彎」，不合理（房價不會變大反而變便宜）
				- 三次：更靈活，能在後段再往上彎，比較符合直覺，可能 overfit 
			- 偏差–變異權衡（bias–variance tradeoff）
				- 當模型太簡單時 → **偏差高（High Bias）**
				- 當模型太複雜時 → **變異高（High Variance）**
		- Feature Scaling 變重要
			- $x$ 範圍：1 ~ 1000
			- $x^2$ 範圍：1 ~ 1,000,000
			- $x^3$ 範圍：1 ~ 1,000,000,000
			- 這會導致：
				- 特徵尺度差距極大
				- 梯度下降更新會變得非常困難
				- 當加入高次項時，**feature scaling 幾乎是必要的**
	- ![[截圖 2026-02-17 晚上9.24.32.png]]
		- Feature Engineering 是機器學習成敗關鍵之一
			- 可以加入「任何你認為合理的轉換」，不一定只能用多項式
			- 為什麼 √x 可能合理？
				- 前段上升快
				- 後段趨緩
				- 不會下降
			- 如何決定用哪些特徵？
