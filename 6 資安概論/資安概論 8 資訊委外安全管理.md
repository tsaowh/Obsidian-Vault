- 資訊委外之相關法規
	- 《資通安全管理法》第9條
		- 公務機關 或 特定非公務機關，於本法適用範圍內，委外 辦理資通系統之建置、維運或資通服務之提供，應考量 受託者 之專業能力與經驗、委外項目之性質 及 資通安全需求，選任適當之受託者 並 監督其資通安全維護情形
	- 《資通安全管理法施行細則》第4條第1項
		- 詳細列出了 委託機關 在選任及監督 受託者時應注意( 從選擇、簽約、執行到終止整個生命週期的資安管理要求 )
			- 資安管理能力
			- 專業人員配置
			- 業務複委託規範
				- 受託者 辦理受託業務 得否複委託
				- 得複委託之 範圍與對象
				- 複委託之受託者 應具備之 資通安全維護措施
			- 業務涉及國家機密
				- 應接受適任性查核，並依《國家機密保護法》之規定，管制出境
			- 安全性檢測
				- 受託者 應提供該資通系統之 安全性檢測證明
				- 委託機關應 自行 或 另行委託第三方 進行安全性檢測
					- 屬委託機關之核心資通系統 或
					- 委託金額達新臺幣一千萬元以上者
				- 涉及 利用 非受託者自行開發之 系統或資源 者
					- 應標示非自行開發之 內容與其來源 及 提供授權證明
			- 資安事件通報及補救
				- 違反資通安全相關法令 或 知悉資通安全事件時，應 立即通知委託機關 及 採行補救措施
			- 委託終止之資料處理
			- 定期稽核
		- 委外後的監督與管理
			- 委託機關如何透過「第二方稽核」來確保委外資安的有效性
			- 稽核優先及判斷標準
				- 委外金額
				- 涉及核心系統
				- 業務性質敏感性
					- 處理大量個人資料、涉及國家機密或關鍵基礎設施
			- 稽核方式彈性
				- 書面稽核
					- 要求廠商提供資安政策、程序文件、訓練紀錄、弱點掃描報告等
				- 現場稽核
					- 對於高風險或有疑慮的案件，進行實地訪查
			- 委託機關 應留存 監督(稽核)結果紀錄、委外廠商之改善報告、以及 後續追蹤紀錄
				- 後續追蹤的重要性
					- 解決才是關鍵
				- 文件化與證據
					- 紀錄是機關履行監督責任的證明
					- 「Check」與「Act」環節
			- 資通系統防護各階段資安強化措施
				- 納入委外契約的附件、技術規範，或作為內部稽核的檢查表
				- 確保 委外廠商 在資通系統的規劃、設計、開發、測試、部署與營運等各階段都能符合資安要求
	- 《資通安全法管理法施行細則》第6條第1項 第 11 款
		- 資通安全維護計畫 應包含 資通系統或服務委外辦理之 管理措施
	- 《資通安全責任等級分級辦法》第11條第2項 
		- 各機關 自行或委外開發之 資通系統 應依 附表九 所定資通系統防護需求分級原則 完成資通系統分級， 並依 附表十 所定資通系統防護基準 執行 控制措施

- 資訊委外之類別及形態
	- 系統發展類(3種形態)
		- 系統開發
		- 系統維護
			- 功能更新、bug修復、效能優化等
		- 系統整合
			- 多個異質系統連接
	- 維運管理類(10種形態)
		- 設備操作
			- 伺服器、網路設備、儲存系統等 
			- 日常監控、狀態管理 與 效能調校
		- 硬體維護
			- 伺服器、網路設備、儲存系統等
			- 定期保養、故障診斷 以及 實體維修
		- 機房設施管理
			- 如電力、空調、消防、門禁等機房環境的管理
		- 備份與復原服務
		- 網路與資安服務
		- 網路管理
		- 資料處理
			- 輸入、整理、分析、輸出
		- 資料登錄
		- 整體委外
			- 整個IT運營部門 或 大部分IT功能 外包
		- 人力支援
			- 廠商提供 IT 技術人員派駐機關
	- 顧問訓練類(6種形態)
		- 顧問輔導
		- 稽核審查
		- 系統稽核
		- 軟體驗證
		- 教育訓練
		- 整體規劃
	- 雲端服務類(3種形態)
		- 軟體即服務(SaaS)
			- 雲端應用軟體，如雲端郵件、辦公軟體
		- 平台即服務(PaaS)
			- 租用 雲端開發 與 部署平台，用於 開發應用程式
		- 基礎設施即服務(IaaS)
			- 租用 雲端運算資源，如虛擬主機、儲存、網路

- 資訊委外之風險
	- 遠端維運之常見資安風險
		- 帳密被竊取
			- 委外廠商的 虛擬私人網路(VPN) 或 其他遠端連取帳號密碼
		- 惡意程式植入
			- 廠商用於維運的主機 本身缺乏足夠的資安防護
		- 弱密碼 或 未限制來源IP
			- 遠端桌面服務(如RDP)
	- 遠端維運應採「原則禁止、例外允許」方式
		- 依循法規並落實管理機制
			- 《資通安全管理法施行細則》第4條
			- 《資通安全責任等級分級辦法》附表十
			- 遠端存取政策、申請流程、審批機制、監控機制、定期審查機制
		- 開放遠端存取期間原則以 短天期 為限，並建立 異常行為管理機制
			- 「短天期」原則
			- 異常行為管理
				- 監控與告警
					- 例如非工作時間登入、來自異常 IP 的連線、大量資料下載、未經授權的指令執行等
				- 即時回應
			- 於結束遠端存取期間後，應確實關閉網路連線，並更換遠端存取通道 (如VPN) 登入密碼
				- 即時關閉
				- 更換登入密碼
				- 多因子鑑別 (MFA)

- 資訊委外之資安要求
	- 一般採購流程可區分以下階段
		- 計畫階段
			- 資安需求明確化，並進行 資安風險評估
		- 招標階段
			- 資安要求 納入 招標文件，並評估 投標廠商的資安能力
		- 決標階段
		- 履約管理階段
		- 驗收階段
		- 保固階段
	- 計畫階段
		- 資訊委外可行性分析
			- 篩選適合委託辦理之業務項目
				- 非所有業務都適合委外
				- 機敏性極高、涉及國家核心利益、或有嚴格法規限制的業務，需審慎評估
				- 或 應保留部分核心功能於內部
			- 進行成本效益分析
				- 委外 是否能引入更專業的資安技術及人才
				- 潛在的資安事件損失
			- 評估資訊委外資安風險與對策
		- 資訊委外專案編成
			- 確立 專案組織架構 及 資源配置，確保 資安專業力量能有效參與
		- 資訊委外資安需求識別
			- 建立資訊委外資安策略
				- 視專案時程定期審查資安策略
					- 資安策略並非一成不變
					- 重大營運變革、新法規發布、架構調整、政策或契約變更等
				- 確保資安策略議題相關人員知悉(包含委外廠商相關人員)
			- 識別委外廠商之限制
				- 委外業務的敏感性及法規要求，對潛在廠商設定資格限制
			- 邀請廠商提出對應措施方案
				- RFI 或 RFC
					- RFI
						- 蒐集資料、需求、產品與價格資訊
					- RFC
						- 提出技術規格，供討論、修訂或形成標準
			- 建立資訊委外資安管理計畫
				- 徵求建議書文件(詳細陳述對 欲採購產品 或 服務本身 資安要求)
				- 資訊委外契約書(明訂採購流程中廠商之 資安義務與責任)
				- 資訊服務水準協定(明訂可量測之服務水準)
	- 招標階段
		- 招標階段之重點在選擇 適宜之委外廠商
		- 招標階段作業包含
			- 定義委外廠商評估準則
			- 備妥保密協議書
			- 招標文件
			- 蒐集廠商投標文件及服務建議書等
		- 資安管理重點著重在 RFP撰寫之完整度 與 評選準則中之資安要求符合度
			- 強調RFP的重要性
			- 符合度審查
				- 不僅要看廠商是否「會做」，更要看他們「如何做」，以及是否有具體的 執行計畫和承諾
		- 招標文件之 RFP及契約書，應依 委外服務之資通系統等級，納入 資通系統防護基準之 系統發展生命週期相關規定
			- 包括需求階段、設計階段、開發階段、測試階段、部署與維運階段、委外階段
	- 決標階段
		- 決標階段之重點為 與廠商之簽約作業
			- 機關與委外廠商 雙方應依據 機關招標文件與廠商回應之服務建議書 進行最終協議，於雙方協議並確定契約內容後，即進行簽約作業
		- 簽約行為重點
			- 查核廠商是否完成 保密切結
			- 完成專案編成
				- 這是決標後，廠商在正式履約前，需要配合機關完成的「專案啟動準備」
					- 保密與資安措施規劃
					- 廠商專案組織與人員
					- 賦予適當職掌
					- 查核與紀錄
	- 履約管理階段
		- 7 個面向
			- 資通安全組織
				- 機關 與 委外廠商 皆應指定專案管理人員，負責推動、協調及督導資通安全管理事項
			- 資訊委外風險持續識別
				- 資訊委外過程之 資通安全風險，宜在核准廠商存取內部設施前 加以識別，並作適當之控制措施
					- 如:遠端存取、資料傳輸、特權帳號管理等
			- 資通委外人力資源安全(雇用前、期間、終止或變更)
			- 資訊委外實體與環境安全
				- 關鍵或敏感的資訊處理設施
				- 安全屏障與進出控制措施
			- 資訊委外管理
				- 符合機關資通安全政策與程序、資通系統防護基準之要求
			- 資訊委外使用者存取管理
				- 權限的申請、審批、發放、變更、收回都必須有文件紀錄
				- 僅授予完成任務所需的 最小權限; 時間權限應有時效性，到期自動收回
			- 資訊委外資通安全事件管理
				- 應備妥正式的 事件通報與提報程序，提供委外廠商配合並施予教育訓練
	- 驗收階段
		- 一般驗收程序
			- 機關 依據 契約文件與「履約管理」階段執行成果 辦理
			- 勞務驗收，得以 書面或召開審查會 方式辦理
				- 書面驗收文件或審查會紀錄，得視為 驗收紀錄
			- 機關可要求委外廠商於 簽約後一定時間內提交「專案工作計畫書」
				- 確認委外專案之 進行方式、專案組織、相關時程及資通安全要求事項 
			- 委外廠商也須將定期召開工作進度報告會議之內容
				- 如應完成重要工作項目、已完成工作的項目、預計工作項目、問題與建議等
		- 資安驗收內容
			- 顧問訓練類
				- 對於需 使用軟體輔助 才可完成專案之情況，應確認委外廠商是否使用 最適之檢測工具與版本
					- 顧問服務(例如弱點掃描、滲透測試、原碼檢測等)
					- 機關應要求廠商說明所使用的工具名稱、版本
					- 確認這些工具是業界公認的、具備最新漏洞庫的「最適」工具
			- 系統發展類
				- 除 功能與效能測試 外，應要求委外廠商提供該資通系統之 安全性檢測證明
					- 弱點掃描報告
					- 滲透測試報告
					- 原碼檢測報告
				- 資通系統使用 非委外廠商自行開發之元件 時，宜要求委外廠商 揭露第三方程式元件之 來源與授權 證明，以確保其元件 非來自大陸地區 或 其他限制地區
					- 供應鏈安全
						- 現代系統 通常會使用大量的 開源軟體、函式庫或第三方元件
					- 來源與授權
						- 詳細揭露這些第三方元件的清單、來源(例如哪個開源專案、哪個供應商)、授權模式
					- 地域限制
			- 維運管理類
				- 維運管理類與顧問訓練類之服務類似？
					- 資安驗收並非一次性的，而是持續性的
					- 驗收的重點在於其日常營運是否符合資安規範，以及面對資安事件的應變能力
				- 若維運過程 有新發現程式漏洞，則需進行 程式修補 或 定期進行程式弱點掃描
			- 雲端服務類
				- 雲端服務類與系統發展類類似
				- 除確認 功能與效能 外，對於 產品或服務 之資安保證大多來自於 委外廠商所提供之證明
					- 雲端服務的特殊性在於其「責任共擔模型」
					- 雲端供應商 負責 底層基礎設施的安全，而客戶 負責 雲端上資料及應用程式的安全
					- 機關在驗收時，很大程度上需要依賴 供應商提供的資安證明
				- 機關 應確認與評估 雲端服務供應商 宣稱之認證範圍，包含 控制與評估涵蓋功能及服務
					- 供應商的 資安驗證 並非涵蓋所有服務
					- 某個雲端供應商可能其 IaaS 通過 ISO 27001，但不代表其 SaaS 服務也通過相同驗證
		- 專案結束後之處置
			- 專案結束或廠商人員離職後 立即停用或移除
				- 實體存取權限(如門禁卡、機房鑰匙) 
				- 邏輯存取權限(如系統帳號、VPN 權限、特權帳號)
			- 屬於機關之資訊資產 回收與銷毀
	- 保固階段
		- 保固服務
			- 保固期間 不論軟硬體資產，應以維持 驗收完成時之狀態 為主要目的
			- 廠商有責任 在保固期內 修補缺陷、處理漏洞，並維持系統的穩定與安全
		- 異常管理
			- 保固期間運作中之 資訊處理設施與應用軟體系統，均應受到嚴格之 變更管理控制
				- 每一次變更都可能引入新的漏洞或風險
				- 遵循機關既定的變更管理程序，包括變更申請、資安評估、核准、測試、實施及記錄。
			- 保固期間系統 如有委外廠商派駐人員 協助者，發生異常事件時，應由派駐人員 負責 將問題反映至資訊業務承辦人員，再循正常程序陳報
